# 1. 문제 정의

## 1) 기존 프로세스
1. 고객으로부터 메뉴를 구성할 각 항목의 이름과 부가 설명, 가격을 옮겨 적고 표로 정리합니다.
2. 정리된 표를 바탕으로 Figma를 사용하여 메뉴 이미지를 제작합니다.
    - <img src="https://github.com/KimRass/KimRass/assets/67457712/2b8ea39c-d8cf-4ea9-a956-a5fa678c2554" width="400">
3. 제작한 메뉴 이미지를 이미지 번역합니다. 즉 바운딩 박스 생성 → 텍스트 인식 → 텍스트 제거 → 번역 → 텍스트 렌더링을 순차적으로 진행합니다.

## 2) 문제점
- 메뉴를 옮겨 적을 때 이미 텍스트를 인식하였고, 메뉴 이미지를 제작할 때 이미 텍스트의 위치를 알고 있음에도 불구하고 불필요하게 텍스트 인식 작업을 수행합니다.
- 메뉴 이미지 제작 과정에서 수작업이 필요합니다.
- 텍스트 제거할 것임에도 불구하고 불필요하게 렌더링합니다.
- 이미지 번역 결과물의 시각적 품질이 매우 낮습니다.
    - <img src="https://github.com/KimRass/KimRass/assets/67457712/6ab9a3d5-8ddd-4db3-96ac-1b5dbd181e9f" width="400">

# 2. 문제 해결

## 1) 메뉴 이미지 생성 자동화
- 메뉴 각 항목의 이름과 부가 설명, 가격을 엑셀 파일로 입력 받아 메뉴 이미지를 자동으로 생성하는 기능을 개발했습니다. 이때 텍스트 렌더링 정보는 이미지로 만들지 않고 메뉴 이미지와 함께 백 엔드에 전달합니다.
- 고객의 요구사항에 맞게 메뉴 항목별 폰트 크기, 블록 높이, 배경 이미지 등을 설정할 수 있도록 했습니다.
- 텍스트 속성 인식 모델과 마찬가지로, 언어별 특성을 고려해 텍스트의 줄바꿈 위치를 계산하도록 했습니다. 즉 언어별로 줄의 시작 또는 끝에 올 수 없는 문자와 같은 규칙이 있는데 [1], 이에 위배되지 않도록 정교한 문자열 처리 알고리즘을 개발했습니다.

## 2) 개선된 프로세스
1. 고객으로부터 메뉴를 구성할 각 항목의 이름과 부가 설명, 가격을 옮겨 적고 표로 정리합니다.
2. 정리된 표를 바탕으로 번역을 진행합니다.
3. 자동화된 기능을 통해 메뉴 이미지를 생성하고, 프론트 엔드에서는 텍스트 렌더링 정보를 전달 받아 텍스트를 렌더링합니다.
    - <img src="https://github.com/KimRass/KimRass/assets/67457712/269b5620-2607-46fc-9579-64a4d6dd46b3" width="400">

# 3. 성과
- Figma를 통한 메뉴 이미지 제작 과정이 제거됐습니다.
- 불필요하고 중복된 작업이 없습니다.
- 불필요한 바운딩 박스 생성, 텍스트 제거, 텍스트 인식 작업을 수행하지 않아도 됩니다.
- 이미지의 시각적 품질이 대폭 향상됐습니다.
