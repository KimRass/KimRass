# 1. 문제 정의
- 메뉴 이미지의 특성상 텍스트간의 거리가 매우 가까운 경우가 많고
- 번역 및 텍스트 렌더링을 어떤 단위로 할 것인가가 중요함
- 그런데 이걸 판단하는 건 매우 미묘하기 때문에 수작업으로 해 왔음
- 이를 자동화하기 위함.
- 바운딩 박스 생성이 첫 번째 프로세스이며 이것이 완료되어야 그 뒤의 다른 모든 프로세스가 진행 가능.
- 바운딩 박스 생성 속도가 병목으로 작용.
- 바운딩 박스의 품질이 전체적인 이미지 번역의 품질과 직결.

# 2. 문제 해결

## (1) 데이터 레이블링
- 바운딩 박스 및 메뉴 이미지 약 8,000장 존재.
- 축적된 데이터에 있어서 일관된 규칙성이 없어 모델 학습 시 모델이 헷갈려 할 것 우려
- 모델 학습이 용이하도록 일관된 법칙을 정해 바운딩 박스 수정
- 유형:
    - 바운딩 박스가 텍스트보다 너무 큼. (그러나 텍스트가 없는 모델이 알아서 텍스트가 있는 영역만 학습함)
    - 바운딩 박스간 겹침
    - 바운딩 박스로 텍스트를 완전히 나눌 수 없는 상황 존재.
    - 바운딩 박스가 그와 무관한 텍스트를 침범.
    - 바운딩 박스가 텍스트 렌더링된 공간을 벗어남.
    - 하나의 의미를 가진 텍스트를 여러 개의 바운딩 박스로 분할.

## (2) 모델 선정

### 1) 'PixelLink' 선정
- 모델이 가정하는 상황이 우리의 문제 상황과 일치한다고 판단.
- 픽셀별 텍스트 여부 분류. 그러나 이것만으로는 서로 구분되어야 하는 텍스트가 하나로 인식되는 문제.
- 정확한 구현체를 찾을 수 없어 일부는 가져오고 일부는 직접 구현.
