# 바운딩 박스 생성 자동화 모델 개발

# 1. 문제 정의
- 이미지 번역에서 바운딩 박스 생성은 가장 먼저 이루어지는 작업입니다.

- 이 작업의 목적은, 이후 텍스트를 번역 및 렌더링하기 쉽도록 적절하게 의미 단위로 나누는 것입니다. 바운딩 박스 생성이 완료되어야만 이후의 다른 모든 작업이 진행 가능하며, 바운딩 박스의 품질이 이미지 번역의 전체적인 품질을 좌우하므로 아주 중요한 단계입니다.
- 수작업을 조금이라도 줄이기 위해, 기존에는 Google Cloud Vision API의 OCR 기능을 사용하여 1차적으로 바운딩 박스를 생성한 다음, 사람이 보정 작업을 수행했습니다. 보정 작업은 둘 이상의 바운딩 박스를 합치거나 하나의 바운딩 박스를 둘 이상으로 나누는 등의 과정입니다. 하지만 의미 단위로 나눈다는 것이 정말 미묘하며, 메뉴의 특성상 텍스트간의 거리가 가까운 경우가 많기 때문에, Google Cloud Vision API도 보정 작업이 많이 필요해 성능에 한계가 있었습니다.
- 이를 해결하기 위해, 이미지의 시각적인 요소를 바탕으로 텍스트를 의미 단위로 적절하게 나누고 텍스트에 딱 맞는 바운딩 박스를 자동 생성하는 모델을 개발하고자 기획했습니다.

# 2. 문제 해결

## (1) 데이터 레이블링
- DB에 존재하는 1만 장 이상의 이미지 중, 중복되거나 너무 단순한 이미지를 제외하고 모델 학습 적합한 8,000여 장 추려냈습니다.
- 바운딩 박스 생성 정책이 지속적으로 변화해 왔고, 작업자마다 각기 다른 스타일로 작업을 해왔기 때문에, 생성된 바운딩 박스에서 일관된 규칙이 없었습니다.
- 잘못된 바운딩 박스 유형:
    - 바운딩 박스가 텍스트에 딱 맞지 않고 너무 큼.
    - 바운딩 박스가 누람됨.
    - 불필요한 바운딩 박스가 존재함.
    - 바운딩 박스간 겹침.
    - 분할이 필요한 텍스트에 하나의 바운딩 박스만 존재.
    - 하나의 의미를 가진 텍스트를 여러 개의 바운딩 박스로 분할.
- 이러한 데이터를 그대로 모델 학습에 사용할 경우, 모델이 제대로 수렴하지 않거나 유용하지 성능이 낮은 모델이 만들어질 것이 우려됐습니다. 이에 따라, 일관된 규칙을 기반으로 모든 이미지의 바운딩 박스 수정 작업을 매일 조금씩 진행했습니다.

## (2) 모델 선정

### 1) 'PixelLink' 선정
- 다양한 scene text detection에 관한 논문을 조사하던 중, 'PixelLink'라는 모델이 이 문제 상황과 일친다고 판단했습니다. 이 모델은 두 가지의 픽셀 단위 추론을 수행합니다. 먼저 'text'/'non-text'의 2가지 클래스로 분류를 수행합니다. 'text'로 분류된 픽셀에 한해서 8가지 방향에 대해서 다른 'text' 픽셀과 연결되어 있는지를 추론합니다. 어느 한쪽 이상이 연결되어 있다고 추론되면 두 픽셀은 서로 동일한 텍스트로 분할됩니다. 픽셀별 텍스트 객체에 대한 예측 결과를 바운딩 박스로 변환하면 원하는 결과를 얻을 수 있습니다. 논문에서는 이러한 방식으로 가까운 텍스트도 분할할 수 있다고 주장합니다.
- 조사를 통해서 이 모델의 비공식 구현체를 몇 개 찾을 수 있었지만 어느 쪽도 완전히 잘 작동하지는 않았습니다. 따라서 구현체를 최대한 이용하되 부족한 부분은 직접 구현하고 직면한 문제를 해결하기 위한 코드를 개발했습니다.

# 3. 성과
- 바운딩 박스 수정 작업에 예상보다 많은 시간이 소요되어, 500여 장밖에 완료하지 못했습니다.
- 수정된 바운딩 박스를 포함하여 모든 데이터를 가지고 모델을 학습시켜본 결과, 모델이 텍스트 객체를 분류할 수 있으며 이를 쉽게 바운딩 박스로 변환 가능하다는 것을 확인했습니다.
